# Техническое задание на проект: «Junior Analyst Vacancy Monitor»

### 1. Цель проекта и концепция
**Название:** Веб-сервис агрегации вакансий для начинающих аналитиков (Junior) в РФ.

**Проблема:** Начинающему специалисту сложно ориентироваться в тысячах вакансий на HeadHunter. Поисковая выдача часто засорена нерелевантными позициями (кассиры, риелторы), а требования к навыкам и зарплатные вилки размыты.

**Решение:** Сервис автоматически собирает релевантные вакансии через API, очищает данные от мусора, стандартизирует форматы работы и предоставляет аналитическую сводку (востребованные навыки, средние зарплаты, доля удаленки) в едином интерфейсе.

### 2. Роли пользователей
*   **Гость (Соискатель):**
    *   Видит аналитический дашборд на главной странице (графики и метрики).
    *   Ищет вакансии, используя фильтры (Город, Опыт, Формат работы).
    *   Может перейти на HH.ru для отклика.
*   **Администратор:**
    *   Имеет доступ к административной панели Django.
    *   Управляет справочниками навыков и компаний.
    *   Контролирует наполнение базы данных.

### 3. Модели данных (Сущности)
Система построена на реляционной БД (SQLite) и включает 3 ключевые модели:

**1. Employer**
*   `name`: Название компании
*   `hh_employer_id`: Уникальный ID на HeadHunter
*   `url`: Ссылка на профиль компании

**2. Skill**
*   `name`: Название технологии (например, SQL, Python, BPMN)
*   *Используется для построения аналитических графиков.*

**3. Vacancy**
*   `title`: Заголовок вакансии
*   `employer`: Связь с Работодателем
*   `salary_from` / `salary_to`: Зарплатная вилка
*   `currency`: Валюта
*   `city`: Город
*   `experience`: Требуемый опыт
*   `employment_mode`: Формат работы
*   `published_at`: Дата публикации
*   `skills`: Связь с моделью Skill

### 4. Ключевой функционал 
1.  **Сбор и обработка данных :**
    *   Скрипт подключается к API HH.ru.
    *   Фильтрует вакансии по ключевым словам "Analyst/Аналитик" строго в названии.
    *   **Нормализация данных:** Преобразует технические коды HH (`flexible`, `remote`) в понятные форматы работы ("Гибрид", "Удаленно").
    *   **Авто-очистка:** Перед загрузкой автоматически удаляются вакансии старше 10 дней для поддержания актуальности.
2.  **Аналитика и Визуализация:**
    *   Расчет средней зарплаты по рынку.
    *   Визуализация "Топ-10 навыков" (Столбчатая диаграмма Chart.js).
    *   Визуализация "Формат работы" (Круговая диаграмма Chart.js).
3.  **Поиск и Фильтрация:**
    *   Пользователь фильтрует вакансии по Городу, Опыту и Формату.

### 5. Внешние интеграции и стек
*   **API:** Интеграция с открытым API **HeadHunter** (библиотека `requests`). Используется `User-Agent` для обхода блокировок.
*   **Backend:** Python 3.13, Django 6.0.  
*   **Database:** SQLite.
*   **Frontend:** Bootstrap 5 (UI-компоненты), Google Fonts (шрифт Inter).
*   **Visualization:** Библиотека Chart.js.
